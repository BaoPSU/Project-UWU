### Duo Tailsitter Model Created (`simulation/duo_tailsitter_guide.md`)
Complete new model built from scratch, based on standard_vtol (X8 flying wing mesh). Key design decisions:

| Change | Reason |
|--------|--------|
| Based on standard_vtol, not quadtailsitter | X8 mesh matches real airframe visually |
| 2 wing-tip motors, joint axis `1 0 0` (body X) | Body X = world Z when standing nose-up, so thrust = upward |
| Spawn pose `-1.5707963` pitch (-90°) | Rotates body nose to point world-up |
| Collision box Z: 0.05 → 0.3 | Thin default box causes model to fall over when standing |
| Prop mesh rotation -90° around Y | Iris prop disc is XY-plane by default; needs to face body X |
| Motor mount cylinder visuals | Orientation reference, makes motor axis visible |
| Removed rotors 0–3 and rotor_puller | Duo tailsitter uses only 2 wing-tip motors |
| Kept servo_0 and servo_1 (elevons) | Pitch/roll control in both hover and forward flight |

### PX4 Airframe Created (`ROMFS/px4fmu_common/init.d-posix/airframes/4022_gz_duo_tailsitter`)
- `CA_AIRFRAME 4` — tailsitter control allocation
- `CA_ROTOR_COUNT 2` — two wing-tip motors
- `VT_TYPE 0` — tailsitter VTOL type
- `FD_FAIL_P/R 180` — disables attitude failure detector (tailsitter sits at 90° pitch)
- `SYS_HAS_NUM_ASPD 0` — no airspeed sensor in sim
- `COM_RC_IN_MODE 1` — arm/control via QGC without physical RC transmitter
- `EKF2_MAG_TYPE 0` — automatic magnetometer fusion
- Conservative hover rate gains: MC_YAW_P 2.0, MC_YAWRATE_P 0.2

### Bugs Found and Fixed

| Bug | Fix |
|-----|-----|
| Model spawning upside down | Changed spawn pitch from +1.5707963 to -1.5707963 |
| Model falling over when standing | Thickened collision box Z from 0.05 to 0.3 |
| Props visually sideways | Added -90° Y rotation to prop visual and collision |
| Attitude failure detected (90° pitch) | Added FD_FAIL_P 180 and FD_FAIL_R 180 |
| Multiple Gazebo windows | PX4 launches its own GUI — stopped running gz sim -g manually |
| Stale params from previous airframe | Must rm parameters.bson before switching airframes |
| CBRK_AIRSPD_CHK not found | Replaced with SYS_HAS_NUM_ASPD 0 |
| FW_ARSP_MODE not found | Removed (doesn't exist in this PX4 version) |
| COM_ARM_EKF_YAW not found | Removed (doesn't exist); EKF2 convergence is timing-based |
| Gazebo starts paused (lockstep) | Run: `gz service -s /world/default/control ... 'pause: false'` |

### Run Commands (as of this week)

```bash
# Wipe param cache (required when switching airframes)
rm -f ~/PX4-Autopilot/build/px4_sitl_default/src/modules/simulation/gz_bridge/parameters.bson
rm -f ~/PX4-Autopilot/build/px4_sitl_default/src/modules/simulation/gz_bridge/parameters_backup.bson

# Launch PX4 + Gazebo
cd ~/PX4-Autopilot/build/px4_sitl_default/src/modules/simulation/gz_bridge
GZ_DISTRO=harmonic GZ_IP=127.0.0.1 PX4_SIM_MODEL=gz_duo_tailsitter \
  PX4_GZ_SIM_RENDER_ENGINE=ogre \
  ~/PX4-Autopilot/build/px4_sitl_default/bin/px4

# If Gazebo starts paused (sim clock frozen):
gz service -s /world/default/control \
  --reqtype gz.msgs.WorldControl \
  --reptype gz.msgs.Boolean \
  --timeout 2000 \
  --req 'pause: false'

# Launch QGC
~/Applications/QGroundControl_*.AppImage &
```
